ScreenLingo 项目需求梳理
=======================

一、项目定位
-----------
低功耗、非侵入式的 Windows 屏幕实时翻译助手（QT C++）。
核心原则：用户正常操作底层软件时完全不受干扰。


二、四大核心工作模式
--------------------

| 模式         | 行为说明                                          |
|--------------|---------------------------------------------------|
| 实时翻译     | 监测区域画面变化 → 文字出现即翻译，消失即清除      |
| 快照翻译     | 静止待机，快捷键触发时翻译当前单帧                 |
| 暂停翻译     | 停止检测与运算，已有译文冻结显示                   |
| 长按翻译     | 按住快捷键翻译鼠标周围文字，松开瞬间销毁           |

切换方式：快捷键 或 系统托盘菜单


三、关键功能模块
----------------

3.1 区域管理
  - 多区域自由框选（类似截图软件体验）
  - 全屏自适应 + 多显示器 + DPI 缩放适配
  - 已选区域支持移动 / 缩放 / 删除

3.2 视觉自定义（实时预览，无需重启）
  - 文字颜色：调色板选择
  - 背景颜色：调色板选择
  - 背景透明度：0%（全透明）～ 100%（实心）
  - 边框宽度与颜色

3.3 智能防重叠布局（三级避让优先级）
  优先级从高到低：
    1. 绝不遮挡原文区域
    2. 新译文不覆盖已有译文气泡
    3. 不超出屏幕可视边界
  动态调整策略：
    首选右侧 → 空间不足则移至下方/左侧/其他空白区
    极端拥挤 → 自动缩小字号或截断文字

3.4 全局控制
  - 一键全局显示 / 隐藏所有译文（优先级高于暂停状态）
  - 暂停模式时托盘图标变色，防止用户遗忘当前状态


四、非功能需求（性能硬指标）
----------------------------

| 指标       | 要求                                               |
|------------|----------------------------------------------------|
| 静止功耗   | 屏幕静止时资源占用极低                              |
| 暂停功耗   | 后台处理完全停止，资源趋近于零                      |
| 长按响应   | 触发到显示迅速，松开即销毁无延迟                    |
| 内存占用   | 常驻内存保持在较低水平                              |
| DPI 适配   | 不同缩放比切换时译文坐标精准对齐，无错位             |
| 多屏支持   | 识别鼠标所在屏幕，在该屏幕上正常工作                |
| 点击穿透   | 译文窗口不拦截鼠标点击，点击事件穿透到下方应用       |


五、关键交互流程
----------------

5.1 区域选择
  用户触发 → 屏幕蒙层 → 鼠标拖拽框选 → 确认生效

5.2 长按翻译
  按住快捷键 → 识别鼠标位置文字并翻译 → 用户阅读 → 松开快捷键 → 译文立即消失

5.3 样式调整
  打开设置面板 → 调整颜色/透明度/边框 → 屏幕译文即时更新（含暂停冻结的译文）


六、验收标准要点
----------------

  - 暂停模式：资源占用极低，译文冻结不动，调整样式时冻结译文随之变化
  - 长按松开：译文瞬间消失，无残留或延迟
  - 防重叠：原文在屏幕边缘时，译文自动跳转至不遮挡原文的位置
  - 透明度：全透明仅见文字无背景色块；实心完全遮盖背景
  - DPI：高分屏（150%缩放）下译文精准无偏移模糊
  - 交互穿透：点击译文位置，触发的是下方应用程序按钮


七、技术要点（结合QT技术栈）
---------------------------------

  1. 点击穿透  → Qt::WA_TransparentForMouseEvents + WS_EX_TRANSPARENT 窗口属性
  2. 屏幕捕获  → Windows DXGI Desktop Duplication API 低功耗画面抓取
  3. 多区域覆盖层 → 每个翻译区域需要独立顶层无边框窗口
  4. 防重叠布局 → 需实现三级避让自动排版引擎
  5. DPI 感知  → 声明为 Per-Monitor DPI Aware 模式
  6. 全局快捷键 → 系统级热键注册 (RegisterHotKey)
